# Roadmap Management

The roadmap lives at `ROADMAP.md` (project root) and is the source of truth for what to build.

## On session start

1. Read `ROADMAP.md`
2. Summarize current progress and suggest the next valuable step

## During work

- When a task is completed: check off the checkbox `- [x]` in ROADMAP.md
- When the user has a new idea: add it to an appropriate section

## Rules

- **Never remove completed items** — the roadmap is a historical document
- **Checkboxes on everything** — every actionable item must be a checkbox
- **Pick what gives most value now** — no strict ordering
